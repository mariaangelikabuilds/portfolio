- Dark navy background (#0a192f)

### rauno.me — Personality
- OS-style layout with dock navigation
- Interface sounds on interaction
- Side-scrolling project feed
- Photography section: full-screen grid
- Atmospheric abstract imagery
- "Craft" as a core concept
- Editorial typography, confident spacing

**Steal:** Interactive elements, typography confidence, the feeling that the portfolio IS the work.

## Fusion Direction

Brittany's recruiter-friendly structure + Rauno's craft-first personality.
Dark theme (deep navy, not pure black).
Fixed left panel with identity.
Big editorial hero typography.
One bold interactive element (cursor-reactive, ambient, or motion-based).
Project cards that feel designed, not templated.

## Typography Direction

- Display/hero: large, editorial, confident. Consider Inter, Satoshi, or General Sans.
- Body: clean sans-serif, readable at small sizes.
- Monospace accents: for tech stack, code snippets. JetBrains Mono or IBM Plex Mono.
- Self-host all fonts (no Google Fonts CDN). Put in public/fonts/.

## Color Direction

Base palette (starting point, refine in browser):
- Background: deep navy (#0a192f) or near-black (#0f0f0f)
- Text primary: light gray (#ccd6f6) or warm white (#e6e6e6)
- Text secondary: muted (#8892b0)
- Accent: TBD by Angel — candidates: electric blue, acid green, warm orange
- Use accent sparingly: CTAs, active nav, hover states, one hero element

## Case Study Template

Each project page follows this structure:

1. **Hero** — project title, one-line description, hero image/screenshot
2. **Overview** — what it is, who it's for, the problem it solves (2-3 sentences)
3. **What I Built** — the actual work, with visuals (screenshots, recordings, diagrams)
4. **Stack** — tech used, displayed visually
5. **Result / Impact** — metrics if available, or qualitative outcome
6. **Learnings** (optional) — what you'd do differently

Keep it tight. 3 scrolls max per case study. Visual-heavy, not text-heavy.
